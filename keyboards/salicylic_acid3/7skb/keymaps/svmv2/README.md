# svmv2 キーマップ

salicylic_acid3 / 7skb 向けの、**SVM（サポートベクターマシン）ベースのTap/Hold判定**を実装したカスタムキーマップです。

---

## 特徴

- 従来の「一定時間＝Hold」判定を廃止し、**2つの特徴量をSVMで判定**することで誤爆を大幅に削減
- **クロスハンド制約**により、同じ手のキーとの組み合わせを自動的にTap扱いにしてタイピング精度を向上
- **リピートTap**機能で同じキーを素早く連打するとキーリピートが効く
- **Retro Tap**により、Holdとして確定後でも他のキーと組み合わさらずに離した場合はTapに変換
- 最大8つのTap/Holdキーを同時追跡

---

## 動作原理

### SVM判定の2特徴量

```
x: T&Hキーが押されてから離されるまでの時間 (ms)
y: T&Hキーが押されてから次のキーが押されるまでの時間 (ms)

スコア = w_x * x + w_y * y + b
スコア > 0 → Hold
スコア ≤ 0 → Tap
```

各キーグループごとに学習済みパラメータが設定されています。

| キーグループ | w_x | w_y | b | guard |
|---|---|---|---|---|
| Tab, Backspace, Space, Enter | 1000 | -290 | -108677 | 250ms |
| Semicolon, Z | 1000 | -175 | -127712 | 250ms |
| ホームポジション・頻出キー | 1000 | 0 | -250000 | 250ms |
| その他 | 1000 | 0 | -200000 | 200ms |

### 状態機械

各Tap/Holdキーは以下の状態を遷移します。

```
[押下]
  │
  ▼
ST_WAITING ──(SVM確定 or タイムアウト)──► ST_HOLDING
  │                                           │
  │(タップ確定)                               │(他キーとコンボ)
  ▼                                           ▼
(解放・終了)                            ST_RELEASING
                                              │
                                              ▼
                                        (Hold解放・終了)
```

### 判定の優先順位

1. **force_tap** フラグ（クロスハンド制約違反）→ 即Tap確定
2. **x と y が両方確定** → SVMスコアで判定
3. **x のみ確定**（T&H離鍵済み、次キー未入力）→ SVM境界を逆算した動的タイムアウトでTap
4. **y のみ確定**（次キー入力済み、T&H未離鍵）→ SVM境界を逆算した動的タイムアウトでHold
5. **どちらも未確定** → guard時間でタイムアウト

---

## クロスハンド制約

`CROSS_HAND_CONSTRAINT = 1`（デフォルト有効）

Layer-Tapキーと**同じ手**の文字キーが押された場合、T&Hキーを強制的にTapとして処理します。
これにより、片手タイピング中にレイヤーが誤って有効化されるのを防ぎます。

- Row 0〜4 = 左手
- Row 5〜 = 右手

`_LAYER3`・`_LAYER4` はクロスハンド制約の対象外（片手完結操作を許容）。

---

## リピートTap

`TAPPING_TOGGLE_TERM = 200ms` 以内に同じT&Hキーを再タップすると、
押しっぱなしでOSのキーリピートが効く状態になります。

- **KC_SPC** はこの機能から除外（Tap直後にHoldとして多用するため）

---

## Retro Tap

Holdとして確定後、**他のキーと組み合わさることなく**離した場合は、
Holdを解除してTapキーコードを発行します。
これにより、ゆっくり押した場合でもTapの入力が取りこぼされません。

---

## レイヤー構成

| レイヤー | 用途 |
|---|---|
| 0: `_QWERTY` | 通常入力レイヤー |
| 1: `_LAYER1` | Fキー・カーソル・メディアキー |
| 3: `_LAYER3` | RGB設定・ブートローダー |

`_LAYER1` は右下の `MO(_LAYER1)` キーで一時有効化。
`_LAYER3` は `TG(_LAYER3)` でトグル。

---

## 設定パラメータ

`keymap.c` 内の以下のマクロで挙動を調整できます。

| パラメータ | デフォルト | 説明 |
|---|---|---|
| `MAX_ACTIVE_TH` | 8 | 同時追跡できるT&Hキーの最大数 |
| `SVM_BUF_SIZE` | 32 | イベントバッファのサイズ |
| `TAPPING_TOGGLE_TERM` | 200ms | リピートTap判定の時間窓 |
| `HOLD_RELEASE_DELAY` | 30ms | Hold → Release 移行までの猶予時間 |
| `MIN_WAIT_TIME` | 50ms | 待機状態とみなす最低時間 |
| `TAP_NON_OVERLAPPED` | false | 重なりのない連続キー入力を即時Tap扱いにするか |
| `CROSS_HAND_CONSTRAINT` | 1 | クロスハンド制約の有効化（1:ON / 0:OFF） |
| `DEBUG_SVM` | 0 | デバッグログ出力（1:ON / 0:OFF） |
| `TRAINING_LOG` | 0 | 物理キー入力の生ログ出力（1:ON / 0:OFF） |

---

## ビルド方法

```sh
qmk compile -kb salicylic_acid3/7skb -km svmv2
```
